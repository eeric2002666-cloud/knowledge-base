# LLM Wiki 知识库

基于 Karpathy 的 LLM Wiki 模式：不是简单 RAG 索引，而是**增量编译**一个持久、互联的 markdown 知识库。

## 核心理念

| 传统 RAG | LLM Wiki |
|----------|----------|
| 每次查询重新检索 | 知识编译一次，持续更新 |
| 碎片化答案 | 已有的交叉引用 |
| 无积累 | 越用越丰富 |

**关键：Wiki 是持久、复合的产物。每次问答都是知识增长的机会。**

## 存储结构

```
/root/.openclaw/workspace/knowledge-base/
├── README.md           # 入口 + 使用说明
├── index.md            # 📋 内容索引（按类型/标签）
├── log.md              # 📜 时间线（ingest/query/lint）
├── schema.md           # 📐 规范（页面格式、命名惯例）
├── sources/            # 📂 原始源（不可变）
│   └── {年}/
│       └── {类型}/
├── wiki/               # 🧠 LLM 编译的维基
│   ├── entities/       # 实体页（人物/产品/公司）
│   ├── concepts/       # 概念页（原理/方法）
│   ├── summaries/      # 摘要页（源文档摘要）
│   └── synthesis/      # 综合页（跨主题）
└── notes/              # 📝 个人笔记
```

## 核心文件格式

### index.md
```markdown
# 知识库索引

## 实体 (entities/)
- [[Apple Design]] - Apple 设计系统规范 (3 sources)

## 概念 (concepts/)
- [[LLM Wiki]] - Karpathy 的知识库模式

## 源 (sources/)
- 2026-04: Apple Design System, LLM Wiki Pattern
```

### log.md
```markdown
# 知识库日志

## [2026-04-08] ingest | Apple Design System
## [2026-04-08] query | 如何构建落地页
## [2026-04-08] writeback | 落地页实现要点
```

### schema.md
```markdown
# 知识库规范

## 页面格式
- **Type:** knowledge/summary/concept/entity
- **Tags:** #标签
- **Sources:** 关联的源文件
- **Last Updated:** YYYY-MM-DD

## 命名惯例
- 实体用 [[Name]] 格式
- 概念用 [[概念名称]] 格式
```

## 操作流程

### 1. Ingest（摄入）
1. 用户添加链接/文件/笔记/结论
2. LLM 读取内容，提取关键信息
3. **更新 wiki**：创建/更新相关页面
4. **更新 index**：添加到索引
5. **追加 log**：记录摄入

### 2. Query（查询）
1. 用户提问
2. LLM 搜索 wiki（读 index → 读相关页面）
3. 合成答案，**带引用**
4. **Write-back**：将答案写回 wiki（可选！）
   - 有价值的分析 → 新页面
   - 新连接 → 更新相关页面交叉引用

### 3. Lint（健康检查）
定期运行：
- 找矛盾点
- 标记过时的 claims
- 找出孤立页面（无 inbound links）
- 标记缺失页面的重要概念
- 建议新问题/新源

### 4. 对话结论保存
讨论得出结论时：
1. 询问用户是否保存
2. 保存到 `sources/conclusions/`
3. 整合到 wiki（更新相关概念页）
4. 追加 log

## 内容模板

### 源文件模板
```markdown
---
title: {标题}
type: {link/document/video/audio/note/conclusion}
tags: [#{标签}]
source_url: {URL}
added: {YYYY-MM-DD}
---

## 要点提炼
{LLM 提取的核心内容}

## 关联概念
- [[概念1]]
- [[概念2]]
```

### Wiki 页面模板
```markdown
# {主题}

**Type:** concept/entity/summary
**Tags:** #{标签}
**Sources:** [源1](), [源2]()
**Last Updated:** {YYYY-MM-DD}

## 概述
{一句话描述}

## 核心要点
- 要点1
- 要点2

## 关联
- [[相关概念1]]
- [[相关概念2]]

## 回顾
- {YYYY-MM-DD}: {回顾笔记}
```

## 使用指令

| 指令 | 功能 |
|------|------|
| `添加 {内容}` | Ingest 新内容 |
| `搜索 {关键词}` | Query wiki |
| `回顾` / `本周新增` | 查看最近 |
| `整理知识库` | 分类 + 更新索引 |
| `健康检查` | 运行 Lint |
| `列出标签` | 查看所有标签 |
| `保存结论` | 保存当前讨论的结论 |

## Write-back 原则

**重要**：每次有价值的问答都应该考虑写回：
- 分析/对比 → 新页面
- 新洞察 → 更新相关页
- 发现的连接 → 添加交叉引用

这样知识库才真正**通过使用增长**。

## Lint 检查清单

运行 `知识库健康检查` 时：
1. [ ] 矛盾检查：新旧源是否有冲突观点
2. [ ] 过时检查：标记可能被新信息替代的内容
3. [ ] 孤立页：无Inbound 链接的页面
4. [ ] 缺失概念：提到但无专属页的主题
5. [ ] 链接检查：坏链、缺失引用
6. [ ] 建议：可补充的问题/源

---

**核心理念**：LLM 做繁琐的维护工作（总结、交叉引用、保持一致），人类做思考和提问。Wiki 越用越聪明。
